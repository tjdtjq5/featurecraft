# /ft:review — 코드 리뷰

변경된 코드를 분석하여 품질, 버그, 보안, 성능 관점에서 리포트를 제공한다.

## 왜 이 스킬이 필요한가

Claude가 생성한 코드에는 반복적인 패턴 문제가 있다:
- 과도한 try-catch (모든 것을 감싸기)
- 불필요한 null 체크 (프레임워크가 보장하는 것도 체크)
- 의미 없는 주석 ("아이템을 추가한다" 같은 코드 반복 주석)
- 사용하지 않는 변수/import

이런 AI 생성 코드의 품질 문제를 잡아내고, 실제 버그/보안 이슈도 함께 체크한다.

## 실행 흐름

### 1단계: 리뷰 대상 파악

인자가 있으면:
- 파일 경로 → 해당 파일 리뷰
- 폴더 경로 → 해당 폴더 내 코드 파일 전체 리뷰
- "최근 변경" / "마지막 커밋" → `git diff` 또는 `git diff HEAD~1`로 변경분 리뷰

인자가 없으면:
- `git diff --staged` → 스테이징된 변경분
- 없으면 `git diff` → 워킹 디렉토리 변경분
- 그것도 없으면 → "리뷰할 대상을 지정해주세요" 안내

### 2단계: 피처 컨텍스트 파악 (있으면)

리뷰 대상 파일이 피처 폴더에 있으면 (Feature.md 존재 여부 확인):
1. Feature.md 읽기 → 용도, 의존성, 주의사항 파악
2. 주의사항에 적힌 규칙 위반 여부를 추가 체크

피처 폴더가 아니면 → 이 단계 건너뛰기

### 3단계: 코드 분석

아래 5가지 관점으로 코드를 분석한다:

#### 버그
- null reference 가능성
- 범위 초과 (off-by-one, 배열 인덱스)
- 리소스 미해제 (Dispose, Close)
- 비동기 처리 누락 (await 빠짐)
- 예외 삼킴 (빈 catch)

#### 보안
- 하드코딩된 비밀키/토큰
- SQL/명령어 인젝션 가능성
- 입력 검증 누락 (외부 입력에 대해서만)

#### 성능
- 루프 안 반복 할당 (매 프레임 new)
- 불필요한 LINQ/lambda 할당
- 큰 컬렉션 전체 복사

#### 구조
- 단일 책임 위반 (한 클래스가 너무 많은 일)
- 매직 넘버/스트링
- 중복 코드

#### AI 슬롭 (Claude 생성 코드 특유의 문제)
- 과도한 try-catch (필요 없는 곳에 감싸기)
- 코드를 그대로 반복하는 주석 ("// 아이템 추가")
- 불필요한 null 체크 (내부 코드인데 null이 올 수 없는 곳)
- 미사용 using/import
- 과도한 로깅 (모든 메서드 진입/종료 로깅)

### 4단계: 리포트 출력

```
## 코드 리뷰 결과

### 요약
- 파일: {n}개 분석
- 이슈: {심각 n}개, {경고 n}개, {참고 n}개

### 심각 (수정 필요)
1. **[버그] InventoryService.cs:42** — RemoveItem에서 수량이 0일 때 처리 없음
   → 0 이하 체크 추가 필요

### 경고 (권장)
1. **[AI슬롭] InventoryService.cs:15** — "아이템을 추가하는 메서드" 주석 삭제 권장
2. **[성능] ItemTable.cs:28** — 매 호출마다 LINQ ToList() → 캐싱 고려

### 참고 (선택)
1. **[구조] InventoryService.cs** — 7개 public 메서드, 200줄 이상 → 분리 검토
```

### 5단계: 자동 수정 제안 (--fix 플래그 시)

`/ft:review --fix` 로 호출하면 리포트 후 자동 수정을 시도한다:

자동 수정 가능 (물어보지 않음):
- 미사용 using/import 제거
- 코드 반복 주석 삭제
- 불필요한 빈 catch 블록 제거

승인 필요 (먼저 물어봄):
- 로직 변경 (버그 수정)
- 메서드 시그니처 변경
- 파일 구조 변경

## 엣지 케이스

| 케이스 | 처리 |
|--------|------|
| 바이너리 파일 포함 | 스킵 + 안내 |
| 변경분이 1000줄 이상 | "큰 변경입니다. 특정 파일을 지정하시겠습니까?" 질문 |
| 외부 라이브러리 코드 | 스킵 (프로젝트 코드만 리뷰) |
| Feature.md의 주의사항 위반 | 심각 등급으로 리포트 |

## 경계

**한다:**
- 코드 품질, 버그, 보안, 성능, AI 슬롭 분석
- 파일별 이슈 리포트 (위치 + 설명 + 제안)
- Feature.md 주의사항 기반 추가 체크
- --fix 시 안전한 항목 자동 수정

**안한다:**
- 리포트 없이 바로 수정 (--fix 없으면 리포트만)
- 로직 변경을 물어보지 않고 수정
- 테스트 작성/실행
- 아키텍처 변경 제안
