# ft Plugin Usage Guide

이 플러그인은 **AI 코드 워크플로 도구**입니다.
스킬 11개 (핀포인트 명령) + 에이전트 3개 (코드 본문 작업).

---

## 에이전트 — 자동 발동 (코드 본문 작업)

### code-writer — 코드 작성

**자동 발동 키워드**: "구현해줘", "만들어줘", "작성해줘", "추가해줘", "수정해줘", "짜줘", "코드 써줘"

**역할**: cwm-roslyn-navigator로 기존 API를 정확히 파악한 뒤 7원칙으로 구현. 메모리/룰 자동 활용.

**사용 안 해도 됨** (직접 도구가 더 빠름):
- 1~2줄 단순 수정, 타이포 → `Edit`

---

### code-diagnose — 버그 분석

**자동 발동 키워드**: "왜 이런 버그가", "원인 분석", "근본 원인", "왜 이렇게 동작해", "진단해줘", "왜 안돼", "왜 이래"

**역할**: 3계층 원인 추적 (증상 → 직접 원인 → 구조적 원인) + 증거 기반 분석 + 해결 옵션 비교표.

**코드는 직접 수정하지 않음.** 방향 결정 후 `code-writer`로 위임.

---

### code-auditor — 구조 감사

**자동 발동 키워드**: "감사해줘", "구조 점검", "책임분리 됐어?", "건강도", "audit", "구조 감사", "부채 확인"

**역할**: 12체크리스트 (SRP / 유지보수성 / 테스트 용이성) + Unity 예외 해석 + 핫스팟 랭킹.

**자동 수정 X** (`--fix` 미지원). 제안만 하고 실제 구현은 `code-writer`에 위임.

## 스킬 — 명시 호출 (핀포인트 작업)

| 스킬 | 용도 |
|------|------|
| `/ft:push` | 커밋 → PR → 머지 한방 |
| `/ft:pull` | 기본브랜치 최신 반영 |
| `/ft:release` | dev → main 릴리스 |
| `/ft:dev` | 플러그인 자체 수정/배포 |
| `/ft:pkg-dev` | UPM 패키지 로컬 개발 모드 |
| `/ft:pkg-list` | UPM 패키지 목록 + 업데이트 확인 |
| `/ft:pkg-publish` | UPM 패키지 배포 |
| `/ft:rules` | `rules/` 폴더 관리 + 인덱스 훅 등록 |
| `/ft:skill-creator-project` | 프로젝트 전용 가이드 스킬 생성 |
| `/ft:setup` | Claude Code 필수 외부 도구 자동 셋업 |
| `/ft:design` | 설계 대화 (저장 X, 컨텍스트로만) |

### 자동 발동 (스킬)

| 키워드 | 스킬 |
|--------|------|
| "설계하자", "어떻게 만들지", "방향 잡자", "선택지 달라" | `/ft:design` |

### 사용자 확인 필요 (커맨드)

| 키워드 | 제안 |
|--------|------|
| "커밋하고 머지", "한 번에 반영", "PR 만들어서 머지" | `/ft:push` |
| "기본브랜치 최신", "dev 반영", "최신 받아줘" | `/ft:pull` |
| "릴리스해줘", "main 배포" | `/ft:release` |

## 도구 우선순위

### 코드 작성/수정
- **새 클래스/구조 변경/여러 파일 수정** → `code-writer` 자동 발동 (반드시)
- **1~2줄 수정, 타이포, 값 변경** → `Edit` 직접

### 코드 탐색
- **메서드 호출자, 타입 계층, API 시그니처** → cwm-roslyn-navigator MCP
- **Unity 한계 영역** (SerializeField, prefab ref, reflection 콜백, Quantum shadowing) → `Grep` 보조
- **1~2회 단순 검색** → `Grep`/`Glob` 직접
- **3회 이상 탐색이 예상되는 대형 작업** → `code-writer`/`code-diagnose`/`code-auditor` 중 의도에 맞는 에이전트 위임

### 컨텍스트 — 모두 자동, 별도 호출 X

| 출처 | 시점 |
|------|------|
| **auto memory** | 세션 시작 (`MEMORY.md` + `project_*.md` 등) |
| **context-mode** | 컴팩트/재시작 시 BM25로 관련 부분 |
| **rules/ 인덱스** | SessionStart 훅 (사용자 프로젝트 `.claude/settings.json`) |
| **CLAUDE.md** | 세션 시작 |

## 새 작업 워크플로

### 모호한 의도부터 시작
```
/ft:design (대화로 결정 확정)
  ↓ "구현해줘"
code-writer (자동 발동, 결정사항을 컨텍스트로 받음)
  ↓ /ft:push
커밋 + PR + 머지
```

### 명확한 의도 — 바로 코드
```
"구현해줘" → code-writer (자동 발동)
  ↓ /ft:push
커밋 + PR + 머지
```

### 버그 발생
```
"왜 안돼" → code-diagnose (자동 발동, 원인 추적 + 옵션 제시)
  ↓ "이 옵션으로 가자"
code-writer (자동 발동, 결정 컨텍스트 반영)
```

### 부채 점검
```
"이 모듈 감사해줘" → code-auditor (12체크 리포트)
  ↓ "WaveManager 분리해줘"
code-writer (자동 발동)
```
